# Documentation

English reports for the CD4⁺ T-Cell Perturb-seq drug-target discovery project. **Start with `FULL_REPORT.md`** — it consolidates every phase, the methods, all figures, and an explicit boundaries/confidence section.

| File | Contents |
|---|---|
| **`FULL_REPORT.md`** | **Consolidated report (Phases A–G)** — read this first. Executive summary, data, methods, per-phase results, boundaries, deliverables. |
| `PROJECT.md` | Objective and running status of the project. |
| `ANALYSIS_PLAN.md` | Detailed scientific plan: phases, discovery directions, deliverables, methods. |
| `IMMUNOLOGY_PRIMER.md` | Background primer for readers new to CD4⁺ T-cell immunology and how to read this screen. |
| `PHASE_A_RESULTS.md` | Foundation: QC/schema audit, reproducibility landscape, positive-control benchmark. |
| `PHASE_B_RESULTS.md` | Six discovery directions (cytokine / polarization / context / network / GWAS / druggability). |
| `PHASE_C_RESULTS.md` | Integration into the composite target scorecard. |
| `TARGET_REPORT.md` | Phase B/C target dossiers with embedded figures. |
| `PHASE_D_RESULTS.md` | Single-cell validation of the 7 flagship targets (donor D4). |
| `PHASE_E_RESULTS.md` | Cross-donor reproduction (4 donors) + novel chromatin/TF-axis validation (19 targets). |
| `PHASE_E_METHODS.md` | Methods reference for the Phase E single-cell pipeline. |
| `PHASE_F_RESULTS.md` | Phase F translational readiness — cross-track ranking table for the 19 targets (structure, chemistry, tractability, clinical, genetics). |
| `TARGET_DOSSIERS.md` | Phase F translational dossiers — full narrative per-target dossiers (companion to the table above). |
| `PHASE_G_RESULTS.md` | Phase G deep single-cell analysis — Tier 1 (dose-response, responder fraction, baseline sensitivity). |
| `PHASE_G_TIER2_RESULTS.md` | Phase G Tier 2 — cross-donor differential abundance, rare-state safety, co-expression rewiring. |
| `PHASE_G_TIER3_RESULTS.md` | Phase G Tier 3 — activation-trajectory checkpoint, dual-guide concordance. |

Figures referenced in these reports live under `../results/<phase>/` and are linked with relative paths, so they render directly on GitHub.

> A Chinese-language counterpart set (`zh/`) exists locally but is excluded from the repository via `.gitignore`.
